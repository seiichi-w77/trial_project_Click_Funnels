# C11: Larkセットアップ

## C11.1: Lark上にデータベースまたは管理スペースを構築

### Lark Baseの概要

**Lark Base**は、統合ワークスペースツール「Lark」のデータベース機能です。
以下の6つの管理項目について、データベースを構築します。

---

## 管理項目1: コンテンツ管理

### 目的
SNS投稿・note記事・ステップメールなど、全てのマーケティングコンテンツを一元管理

### データベース構造

**テーブル名**: `Content_Management`

| フィールド名 | データ型 | 説明 |
|------------|---------|------|
| content_id | テキスト（主キー） | コンテンツID |
| title | テキスト | タイトル |
| content_type | 選択（単一） | タイプ（X投稿/Threads/note/メール/LP） |
| platform | 選択（単一） | プラットフォーム（X/Threads/note） |
| status | 選択（単一） | ステータス（下書き/レビュー中/公開済） |
| scheduled_date | 日時 | 公開予定日時 |
| published_date | 日時 | 実際の公開日時 |
| content_body | 長文テキスト | 本文 |
| hashtags | テキスト | ハッシュタグ |
| images | 添付ファイル | 画像 |
| url | URL | 公開URL |
| impressions | 数値 | インプレッション数 |
| engagement_rate | 数値 | エンゲージメント率（%） |
| clicks | 数値 | クリック数 |
| notes | 長文テキスト | メモ |

### ビュー設定

**ビュー1: 投稿カレンダー**
- カレンダー形式で`scheduled_date`を表示
- 1週間/1ヶ月の投稿予定を可視化

**ビュー2: プラットフォーム別**
- X、Threads、noteごとにグループ化
- ステータス別に色分け

**ビュー3: パフォーマンス分析**
- エンゲージメント率順にソート
- インプレッション数を棒グラフで表示

---

## 管理項目2: リスト（見込み客）管理

### 目的
全ての見込み客を一元管理し、ステージごとに適切なアクションを取る

### データベース構造

**テーブル名**: `Lead_Management`

| フィールド名 | データ型 | 説明 |
|------------|---------|------|
| lead_id | テキスト（主キー） | リードID |
| name | テキスト | 氏名 |
| email | メール | メールアドレス |
| phone | 電話番号 | 電話番号 |
| company_name | テキスト | 会社名・屋号 |
| industry | 選択（単一） | 業種 |
| lead_source | 選択（単一） | 流入元（Meta広告/Google広告/SNS/紹介） |
| registration_date | 日付 | 登録日 |
| stage | 選択（単一） | ステージ（リード/育成中/相談予約済/相談済/成約/失注） |
| last_action_date | 日付 | 最終アクション日 |
| next_action | テキスト | 次のアクション |
| consultation_scheduled | 日時 | 個別相談予約日時 |
| tags | 複数選択 | タグ（engaged_high/pain_leadgen等） |
| email_open_rate | 数値 | メール開封率（%） |
| email_click_rate | 数値 | メールクリック率（%） |
| notes | 長文テキスト | メモ |

### ビュー設定

**ビュー1: ステージ別パイプライン**
- カンバン形式でステージごとに表示
- ドラッグ&ドロップでステージ移動

**ビュー2: 今週のアクション**
- `next_action`が「今週」のリードを表示
- 優先度順にソート

**ビュー3: 流入元別分析**
- 流入元ごとにグループ化
- 成約率を計算

## Lark Baseセットアップ手順

### Step 1: Larkアカウント作成
1. https://www.larksuite.com/ にアクセス
2. 「無料で始める」をクリック
3. メールアドレスで登録

### Step 2: ワークスペース作成
1. 「新規ワークスペース」をクリック
2. ワークスペース名: 「Funnel DX」
3. メンバー追加（必要に応じて）

### Step 3: Lark Base作成
1. 左サイドバー → 「Base」をクリック
2. 「新規Base」をクリック
3. Base名: 「セールスファネル&DX管理」

### Step 4: テーブル作成
上記の6つのテーブルを順番に作成:
1. Content_Management
2. Lead_Management
3. Customer_Management
4. Cash_Flow_Management
5. Resource_Management
6. KPI_Monitoring

各テーブルで、上記のフィールドを設定。

### Step 5: ビュー作成
各テーブルで、上記のビューを作成。

### Step 6: 自動化設定（オプション）
- ClickFunnelsからZapier経由でLead_Managementに自動同期
- 新規顧客獲得時、Slackに通知

### Step 7: アクセス権限設定
- オーナー: 全てのテーブルに編集権限
- メンバー: 必要なテーブルのみ閲覧/編集権限

# C12: スケーリング（事業拡大）戦略

## C12.1: パートナー（アフィリエイト）プログラムの制度設計

### プログラム名
**「Funnel DX パートナープログラム」**

### 目的
- 紹介による新規顧客獲得の拡大
- パートナーとのWin-Win関係構築
- ブランド認知度の向上

---

### パートナー種別

#### レベル1: アフィリエイトパートナー

**対象者**:
- 既存顧客
- SNSインフルエンサー
- ブロガー
- 業界関係者

**報酬体系**:
- 紹介1件につき: **5万円キャッシュバック**
- または: 紹介1件につき成約金額の**10%**（どちらか高い方）

**例**:
- スタンダードプラン（55万円）成約: 5.5万円 → 5万円（固定額の方が高い）
- プレミアムプラン（110万円）成約: 11万円（10%の方が高い）
- VIPプラン（198万円）成約: 19.8万円（10%の方が高い）

**参加条件**:
- 既存顧客 or SNSフォロワー1,000人以上
- Funnel DXの理念に共感していること

**サポート**:
- 専用アフィリエイトリンク発行
- 紹介用テンプレート提供（SNS投稿文、メール文面等）
- 月1回のパートナー向けウェビナー

---

#### レベル2: 認定パートナー

**対象者**:
- VIPプラン修了者
- ファネル構築スキルを習得した方
- 月5件以上の紹介実績がある方

**報酬体系**:
- 紹介1件につき成約金額の**20%**
- さらに、紹介したクライアントのVIP継続プログラム加入時: 継続課金の**10%**（毎月）

**例**:
- VIPプラン（198万円）成約: 39.6万円
- そのクライアントがVIP継続（月10万円）加入: 毎月1万円（継続課金）

**参加条件**:
- VIPプラン修了
- 月5件以上の紹介実績
- 認定試験合格

**サポート**:
- 認定バッジ・ロゴの使用権
- 専用ランディングページ提供
- 優先サポート
- 四半期ごとの認定パートナー限定イベント

---

#### レベル3: リセラーパートナー

**対象者**:
- マーケティングエージェンシー
- Webコンサルタント
- システム開発会社

**報酬体系**:
- 自社ブランドで販売可能（ホワイトラベル）
- 売上の**30%**を手数料として当社に支払い
- 残り**70%**がリセラーの収益

**例**:
- VIPプランを198万円で販売 → 当社に59.4万円支払い → リセラーの収益138.6万円

**参加条件**:
- 法人または個人事業主
- 年間10件以上の販売コミット
- 契約締結

**サポート**:
- ホワイトラベル版の提供
- 技術サポート（クライアント対応は当社が担当）
- 営業資料・提案書テンプレート提供
- 月1回の戦略ミーティング

---

### パートナープログラム運用

#### 1. 登録フロー

1. **申請**: 専用フォームから申請
2. **審査**: 2-3営業日で審査
3. **契約**: 契約書に署名（電子署名）
4. **アカウント発行**: 専用ダッシュボードへのアクセス権付与
5. **オンボーディング**: 初回オンボーディングミーティング（Zoom 60分）

#### 2. 紹介フロー

1. **パートナーが紹介**: 専用リンク or 紹介コード経由で見込み客を紹介
2. **当社が個別相談**: 見込み客と個別相談実施
3. **成約**: 成約した場合、パートナーダッシュボードに反映
4. **報酬支払い**: 月末締め、翌月末払い

#### 3. サポート体制

- **パートナー専用Slackチャンネル**: 質問・相談
- **月1回のパートナーウェビナー**: 最新情報・成功事例共有
- **四半期ごとのパートナー表彰**: トップパートナーを表彰・インタビュー掲載

## C12.2: 代理店や紹介者が活動しやすい再販システムの考案

### 再販システム「Funnel DX Reseller Platform」

#### 概要
パートナーが自社ブランドで販売できるホワイトラベルプラットフォーム

---

### 提供するもの

#### 1. ホワイトラベル版プラットフォーム

**カスタマイズ可能な要素**:
- ロゴ
- カラー（ブランドカラーに変更可能）
- 会社名・サービス名
- ドメイン（例: `funnel-solutions.com`）

**変更不可の要素**:
- カリキュラム内容
- 教材・テンプレート
- サポート体制（当社が提供）

#### 2. 営業・マーケティング資料

- **営業資料（PowerPoint）**: 提案用スライド（50ページ）
- **セールスシート（PDF）**: サービス概要・料金表（10ページ）
- **成功事例集（PDF）**: 実際の成功事例（20ページ）
- **LP テンプレート**: ホワイトラベル版LPのHTMLテンプレート

#### 3. クライアント管理ダッシュボード

リセラー専用のダッシュボードで以下を管理:
- 販売実績
- クライアント進捗状況
- 報酬額
- 次回ミーティング予定

#### 4. 技術サポート

- クライアント対応は当社が担当（Zoom・Slack）
- リセラーは営業・成約のみに集中
- 月次レポートを当社が作成し、リセラーがクライアントに提出

---

### 再販価格設定

リセラーは自由に価格設定が可能（推奨価格あり）。

**推奨価格**:
- スタンダード: 70万円（当社への支払い: 21万円、リセラー収益: 49万円）
- プレミアム: 140万円（当社への支払い: 42万円、リセラー収益: 98万円）
- VIP: 250万円（当社への支払い: 75万円、リセラー収益: 175万円）

**柔軟性**:
リセラーが独自のサポートを追加して、さらに高額で販売することも可能。

## C12.3: 顧客ロイヤリティを高めるためのコミュニティ運営方針の策定

### コミュニティ名
**「Funnel DX コミュニティ」**

### 目的
- 顧客同士の交流促進
- 成功事例の共有
- ピアサポート（仲間同士の助け合い）
- ブランドロイヤリティの向上
- 解約率の低下

---

### コミュニティの構成

#### プラットフォーム
**Slack**（または Lark Messenger）

**理由**:
- リアルタイムなコミュニケーション
- チャンネル分けが容易
- 既存ツールとの連携が簡単

---

### チャンネル構成

#### 1. #全体告知
- 運営からの重要なお知らせ
- イベント告知
- 新機能リリース情報

#### 2. #自己紹介
- 新規メンバーの自己紹介
- アイスブレイク

#### 3. #成功事例シェア
- 「今月〇〇件のリード獲得できました!」
- 「初受注取れました!」
- お互いに祝福し合う

#### 4. #質問・相談
- 技術的な質問
- マーケティング戦略の相談
- 運営・メンバーが回答

#### 5. #ファネル設計相談
- LP・メール文面の添削
- ファネル設計のフィードバック

#### 6. #広告運用Tips
- 広告運用のノウハウ共有
- 成果の出たクリエイティブのシェア

#### 7. #雑談
- 仕事の話以外の雑談
- リラックスした交流

#### 8. プラン別限定チャンネル
- #スタンダード限定
- #プレミアム限定
- #VIP限定

#### 9. 業種別チャンネル
- #コーチング業
- #スクール業
- #EC・物販
- #士業

---

### コミュニティ運営方針

#### 方針1: ピアサポートを促進

**アクション**:
- 運営が全ての質問に答えるのではなく、メンバー同士が助け合う文化を醸成
