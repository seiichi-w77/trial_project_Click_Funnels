## 2. 課題提起セクション (Problem Statement)

**見出し:**
### 開発サイクルの「限界」を感じていませんか？

**本文:**
市場のスピードは加速し続ける一方、開発プロセスは複雑化しています。スプリントの遅延、開発者の燃え尽き、そして繰り返される単純作業。イノベーションのアイデアはあっても、それを形にするリソースと時間が常に不足しています。従来の開発手法は、もはや限界に達しているのかもしれません。

---

## 3. 解決策 & 仕組み (Solution & How It Works)

**見出し:**
### Miyabi: アイデアからPRまで、開発を完全自動化

**シンプルな3ステップ:**

1.  **[Step 1] Issueを作成**
    `gh issue create --title "..."`
    あなたのアイデアやタスクをGitHub Issueとして起票します。これがすべての起点です。

2.  **[Step 2] AIエージェントが協調実行**
    21体以上の専門エージェントがIssueを解釈し、安全なGitブランチで並列的にコーディング、レビュー、テストを進行します。

3.  **[Step 3] Pull Requestが完成**
    テストとドキュメントを含む、マージ可能なPull Requestが自動で生成されます。人間は最終確認を行うだけで、開発プロセスが完了します。

## 4. 主要機能 (Key Features)

**見出し:**
### 開発を加速させる、Miyabiの中核機能

**機能リスト:**

*   **GitHub as OS**
    GitHubをオペレーティングシステムとして捉え、開発のすべてをIssueとPRで完結させる革新的なコンセプト。開発ワークフローが劇的にシンプルになります。

*   **自律型マルチエージェントシステム**
    戦略立案からSNS運用までを担う14種のビジネスエージェントと、コーディング専門の7種のコーディングエージェントが協調し、人間のように複雑なタスクを分担・実行します。

*   **インテリジェントLLMルーター**
    Claude 3.5 Sonnetを主力に、タスクの複雑性に応じてGPT-4oやGeminiを自動で最適選択。常に最高の知能を活用し、コストとパフォーマンスを両立させます。

*   **安全な並列実行基盤**
    Git Worktreeをベースに設計されたアーキテクチャにより、複数のIssueを同時に、コンフリクトなく安全に処理。チーム全体の開発スループットを最大化します。
