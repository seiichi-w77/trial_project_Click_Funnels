## 2. Miyabi AGI OSとは？ (10分)

**林 俊介:**
「Miyabiは、単なるコード生成ツールではありません。**『GitHub as OS』** というコアコンセプトに基づいた、完全自律型の開発システムです。」

（スライド1: Miyabiのコンセプト図を表示）

「ご覧の通り、インプットはただ一つ、皆さんが使い慣れた**GitHubのIssue**です。そしてアウトプットは、コード、テスト、ドキュメントがすべて含まれた**Pull Request**。その間のプロセスを、21体以上の専門家AIエージェントが協調して実行します。」

「例えば、仕様を分析する『企画エージェント』、最適な技術を提案する『設計エージェント』、そして実際にコードを書く『コード生成エージェント』やテストを行う『レビューエージェント』。彼らはまるで、超優秀なリモートチームのように、24時間365日、並列でタスクを処理し続けます。」

「このシステムの名前の由来は、日本の美的概念である『**雅（みやび）**』です。洗練されたアーキテクチャ、調和の取れたエージェントの協調、そして優美な問題解決。それこそが我々の目指す開発の姿です。」

「百聞は一見に如かず、です。これから実際に、一つのGitHub Issueが、MiyabiによってどのようにPull Requestに変わるのか、ライブでお見せします。」

## 3. ライブデモ (15分)

**林 俊介:**
「それでは、デモを開始します。シナリオは非常にシンプルです。現在開発中のWebアプリケーションに、ユーザーから要望の多かった**『ダークモード切り替え機能』**を実装します。」

（画面をターミナルとGitHubリポジトリに切り替え）

「まず、私がプロダクトマネージャーになったつもりで、機能要件をGitHubのIssueとして作成します。」

**【手順1: GitHub Issueの作成】**

```bash
# ターミナルでIssueを作成
gh issue create --title "feat: ダークモード切り替え機能の実装" --body "ユーザーがUIのテーマをライトモードとダークモードで切り替えられるように、トグルスイッチを追加する。"
```

「はい、Issueが作成されました。Issue番号は『135』です。通常であれば、ここからエンジニアがこのIssueをアサインし、ブランチを切り、開発を始めます。しかし、Miyabiでは違います。」

**【手順2: Miyabi AGI OSの起動】**

「私がやることは、このIssue番号をMiyabiの司令塔である『コーディネーターエージェント』に伝えるだけです。」

```bash
# ターミナルでMiyabiを起動
miyabi agent run coordinator --issue 135
```

（ターミナルにログが流れ始める。同時に、VOICEVOXのずんだもんの声がPCから再生される）

**ずんだもん（音声）:**
`「コーディネーターエージェント、起動したのだ！課題 #135 の分析を開始するのだ！」`

**林 俊介:**

「こちらがMiyabiの心臓部、『Tmux Orchestra』です。ご覧の通り、コードを生成する`CodeGenAgent`、CSSを記述する`StyleAgent`、そしてテストを作成する`TestGenAgent`が、それぞれ別のGitワークツリー上で**完全に並列で**作業を進めています。これにより、コンフリクトを最小限に抑え、驚異的なスピードを実現します。」
